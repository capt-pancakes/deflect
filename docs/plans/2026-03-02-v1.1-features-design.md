# V1.1 Features Design

**Date:** 2026-03-02
**Source:** product-review.md V1.1 Roadmap
**Execution:** 3 parallel worktrees

---

## Scope

6 features from the V1.1 roadmap + Convex backend:

1. Convex DB as persistence layer (daily scores)
2. Daily Challenge global stats (percentile rank)
3. Improved tutorial (color matching phase)
4. Combo visual escalation (5x/10x/15x effects)
5. Image share card (canvas-rendered)
6. Haptic feedback
7. PWA install prompt

## Worktree Strategy

### WT1: Convex Backend + Global Stats
**Files:** `convex/` (new), `src/convex-client.ts` (new), `src/game.ts`, `src/renderer.ts`, `.env.local`
**Prerequisite:** User must run `npx convex dev` to authenticate and create project.

**Schema:**
- `dailyScores` table: `{ seed: number, score: number, createdAt: number }`

**Server functions:**
- `submitDailyScore(seed, score)` ‚Äî mutation, inserts a row
- `getDailyStats(seed)` ‚Äî query, returns `{ attempts: number, percentile: number }`

**Client integration:**
- `src/convex-client.ts` ‚Äî wraps Convex client, exposes `submitDailyScore()` and `getDailyStats()`
- `game.ts` `onGameOver()` ‚Äî when mode is `daily`, call `submitDailyScore`
- Game over screen ‚Äî show "X players today | Top Y%" for daily mode
- Menu ‚Äî show today's attempt count near DAILY button

**Offline-safe:** If Convex is unreachable, silently skip. Analytics stub stays for non-score events.

### WT2: Tutorial + Combo Visuals
**Files:** `src/tutorial.ts`, `src/game.ts`, `src/renderer.ts`, `src/particles.ts`

**Tutorial color matching phase:**
- After existing Phase 2 (first deflect), add Phase 3
- Spawn a blue signal, show "MATCH THE COLORS!" text
- Wrong port ‚Üí "WRONG PORT!" flash, reset signal
- Correct port ‚Üí "PERFECT!", complete tutorial
- Auto-complete after 8 seconds

**Combo visual escalation:**
- 5x combo: screen edge glow (subtle blue), particle burst rate +50%
- 10x combo: glow intensifies, particle burst rate +100%, brief screen flash
- 15x+: max glow, camera shake pulse on each catch
- Tie into `beatState.visualIntensity` ‚Äî combo level adds a multiplier
- Reset all effects when combo breaks

### WT3: Share Card + Haptics + PWA Prompt
**Files:** `src/share-image.ts` (new), `src/share.ts`, `src/haptics.ts` (new), `src/pwa-prompt.ts` (new), `src/game.ts`, `src/renderer.ts`, `src/audio.ts`

**Image share card:**
- `src/share-image.ts` ‚Äî offscreen canvas (400x600), styled results with neon aesthetic
- Score, stats, color blocks, URL rendered on canvas
- `canvas.toBlob()` for `navigator.share({ files })`, text fallback

**Haptic feedback:**
- `src/haptics.ts` ‚Äî `vibrateOnCatch()` (15ms), `vibrateOnMiss()` (two 10ms pulses), `vibrateOnMilestone()` (40ms)
- Guarded by `navigator.vibrate` check
- Respects mute toggle (muted = no haptics)

**PWA install prompt:**
- `src/pwa-prompt.ts` ‚Äî intercept `beforeinstallprompt`, store deferred prompt
- Show banner after 3rd game_over OR first daily challenge completion
- "Add DEFLECT to home screen" with [INSTALL] button
- Dismiss persisted to localStorage (`deflect_pwa_dismissed`)
- Never show if already in standalone mode

## Merge Order

1. WT2 (Tutorial + Combo) ‚Äî no backend dependency, self-contained gameplay changes
2. WT3 (Share + Haptics + PWA) ‚Äî no overlap with WT2
3. WT1 (Convex) ‚Äî merged last, after user has authenticated; touches game.ts and renderer.ts but in different sections
